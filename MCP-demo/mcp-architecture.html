MCP 协议原理

Model Context Protocol — AI 模型与外部工具的标准化通信协议

1 一句话定义

MCP 是 AI 世界的 USB-C 接口 — 提供统一的 JSON-RPC 协议， 让 LLM 与外部工具/数据源即插即用。

3 协议五层模型

Layer 1 连接层 进程管道 / HTTP / WebSocket
Layer 2 传输层 stdio · SSE · Streamable HTTP
Layer 3 消息层 JSON-RPC 2.0 (request / response / notification)
Layer 4 方法层 tools/list · tools/call · resources/read · prompts/get
Layer 5 能力层 Tools · Resources · Prompts

5 JSON-RPC 2.0 消息格式

请求 Request
// Client -> Server
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "add",
    "arguments": {
      "a": 1,
      "b": 2
    }
  }
}
响应 Response
// Server -> Client
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [{
      "type": "text",
      "text": "3"
    }]
  }
}
通知 Notification (无 id，不期待响应)
{
  "jsonrpc": "2.0",
  "method": "notifications/initialized"
}

6 初始化握手

// Step 1: Client -> Server  能力协商
{
  "method": "initialize",
  "params": {
    "protocolVersion": "2024-11-05",
    "capabilities": {},
    "clientInfo": {
      "name": "my-app",
      "version": "1.0"
    }
  }
}

// Step 2: Server -> Client  能力声明
{
  "result": {
    "protocolVersion": "2024-11-05",
    "capabilities": { "tools": {} },
    "serverInfo": {
      "name": "math-server",
      "version": "1.0"
    }
  }
}

// Step 3: Client -> Server  就绪通知
{ "method": "notifications/initialized" }

8 三大能力模型

Tools 工具

LLM 可调用的函数
查数据库 · 发邮件 · 计算

tools/list -> tools/call
Resources 资源

LLM 可读取的数据
文件内容 · API 响应 · 知识库

resources/list -> resources/read
Prompts 提示

预定义的提示模板
代码审查 · 翻译 · 摘要

prompts/list -> prompts/get
