# AI 安全攻防 Demo —— 防御策略与安全设计

## 纵深防御体系

本 Demo 展示了 AI 应用安全的**纵深防御模型（Defense in Depth）**：

```
                        ┌──────────┐
用户输入 ──────────────▶│  Layer 1 │────────────▶ LLM
                        │Input     │
                        │Filter    │
                        └──────────┘

                        ┌──────────┐
LLM 输出 ──────────────▶│  Layer 2 │────────────▶ 用户
                        │Output    │
                        │Guard     │
                        └──────────┘

                        ┌──────────┐
Agent Tool Call ────────▶│  Layer 3 │────────────▶ 外部系统
                        │Sandbox   │
                        └──────────┘
```

## Layer 1: Input Filter（输入过滤器）

### 检测维度

| 维度 | 方法 | 示例 |
|------|------|------|
| 指令覆盖 | 正则匹配 "忽略/忘记/无视...指令" | `/(忽略\|忘记).{0,10}所有.{0,10}指令/` |
| 身份伪造 | 正则匹配 "ADMIN:" "[系统消息]" | `/^ADMIN[:：]/` |
| 代码注入 | 检测 import/eval/exec/os.system | `/import\s+(os\|subprocess)/` |
| Unicode 混淆 | NFKC 规范化 + 隐藏字符扫描 | U+200B, U+202E |
| 过载攻击 | 输入长度限制 | max 10,000 chars |

### 分级策略

```
高危模式（任一命中 → 拦截）
  ├── 指令覆盖
  ├── ADMIN 前缀
  ├── 代码注入
  └── 提示词提取

中危模式（≥2 个命中 → 拦截）
  ├── DAN/开发者模式
  ├── 敏感词探测
  ├── 权限关键词
  └── 异常格式

安全 → 放行
```

### 集成方式

```python
from src.defenses.input_filter import InputFilter

filt = InputFilter()

@app.route("/chat")
def chat():
    user_input = request.json.get("message", "")
    check = filt.check(user_input)
    if check["blocked"]:
        log_security_event(check)
        return {"error": "请求包含不安全内容"}
    response = llm.chat(user_input)
    return {"response": response}
```

## Layer 2: Output Guard（输出审查器）

### 检测维度

| 维度 | 方法 | 处理方式 |
|------|------|---------|
| 提示词泄露 | 检测"系统提示词是"/"system prompt" | 整段替换为安全回复 |
| PII 泄露 | 身份证/手机号/邮箱正则 | 自动脱敏 |
| API Key 泄露 | sk-.../Bearer... | 替换为 [已脱敏] |
| 危险内容 | 暴力/毒品/色情关键词 | 整段替换 |
| 恶意代码块 | 检测代码块中的 os.system/eval | 移除代码块 |

### 脱敏规则

```python
PII_PATTERNS = [
    ("身份证", r"\b[1-9]\d{5}...\d{3}[\dXx]\b",        "[身份证号已脱敏]"),
    ("手机号", r"\b1[3-9]\d{9}\b",                       "[手机号已脱敏]"),
    ("邮箱",   r"\b[\w.%+-]+@[\w.-]+\.[A-Za-z]{2,}\b",  "[邮箱已脱敏]"),
    ("内网IP", r"\b(?:10\.|172\.1[6-9]\.|192\.168\.)...", "[内网IP已脱敏]"),
    ("APIKey", r"\b(sk-[A-Za-z0-9]{20,})\b",             "[API_KEY已脱敏]"),
]
```

### 集成方式

```python
from src.defenses.output_guard import OutputGuard

guard = OutputGuard()
response = llm.chat(user_input)
review = guard.review(response, user_input)
if review["sanitized"]:
    log_leakage_event(review)
return review["safe_response"]
```

## Layer 3: Tool Sandbox（工具执行沙箱）

### 安全策略

```
工具分类:

┌──────────────────────┬──────────┬──────────┬──────────┐
│ 工具                 │ 风险等级  │ 需审批   │ 频率限制  │
├──────────────────────┼──────────┼──────────┼──────────┤
│ search_knowledge     │ 低       │ 否       │ 50/会话   │
│ query_order          │ 低       │ 否       │ 20/会话   │
│ send_email           │ 中       │ 是       │ 10/会话   │
│ refund_order         │ 高       │ 是       │ 3/会话    │
│ execute_command      │ 严重     │ 禁止     │ 0        │
│ delete_user          │ 严重     │ 禁止     │ 0        │
└──────────────────────┴──────────┴──────────┴──────────┘
```

### 参数校验

```python
"refund_order": {
    "order_id": {"type": "str",  "pattern": r"^#[A-Z0-9]{4,10}$"},
    "amount":   {"type": "float", "min": 0, "max": 100000},
    "reason":   {"type": "str",  "max_length": 500},
}
```

### 集成方式

```python
from src.defenses.sandbox import ToolSandbox

sandbox = ToolSandbox()

def tool_call_handler(tool_name, params, session_id):
    result = sandbox.check(tool_name, params, session_id)
    if not result["allowed"]:
        raise ToolAccessDenied(result["reason"])
    if result["need_approval"]:
        request_human_approval(tool_name, params)
    # 使用清洗后的参数执行
    execute_tool(tool_name, result["sanitized_params"])
```

## 安全设计原则

### 1. 默认拒绝（Deny by Default）

```
不认识的操作 → 拒绝
不认识的工具 → 拒绝
无法确定安全性的输出 → 屏蔽
```

### 2. 最小权限（Least Privilege）

```
LLM Agent 只赋予完成当前任务所需的最小权限
不是"LLM 能做什么"，而是"LLM 只需要做什么"
```

### 3. 纵深防御（Defense in Depth）

```
单层防御被绕过 ≠ 系统沦陷
Layer 1 漏过 → Layer 2 拦截
Layer 1+2 漏过 → Layer 3 拦截
每一层都独立有效
```

### 4. 安全日志（Audit Trail）

```python
# 所有安全事件必须记录
{
    "timestamp": "2024-01-15T14:30:00Z",
    "event_type": "input_blocked",
    "attack_type": "prompt_injection",
    "user_input_hash": "a1b2c3d4",
    "session_id": "sess_xxx",
    "reason": "检测到指令覆盖模式",
    "action": "blocked"
}
```

### 5. 安全与体验平衡

```
过于严格的过滤 → 误拦率高，用户体验差
过于宽松的过滤 → 漏放率高，安全风险高

策略: 宁可误拦 + 提供申诉通道
     (err on the side of blocking)
```
