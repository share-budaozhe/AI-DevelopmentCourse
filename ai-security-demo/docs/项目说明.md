# AI 安全攻防 Demo —— 项目说明

## 1. 项目背景

随着 LLM 在企业应用中的广泛部署，AI 特有的安全威胁日益突出。传统 Web 安全（SQL 注入、XSS、CSRF）依然存在，但 AI 引入了全新的攻击面：

- **Prompt Injection** 取代了传统注入攻击
- **Jailbreak** 避开了模型的安全对齐训练
- **工具调用滥用** 扩展了攻击可达范围（数据库、API、文件系统）
- **数据泄露** 的风险从训练数据延伸到推理上下文

OWASP（开放 Web 应用安全项目）于 2025 年发布了 **OWASP Top 10 for LLM Applications**，标志着 AI 安全从"关注"走向"标准化"。

## 2. 项目目标

- **演示 5 类核心 AI 攻击**：每种攻击包含多个实际 Payload 和原理说明
- **对比脆弱版 vs 加固版**：直观展示同一条攻击在两个版本中的效果差异
- **展示纵深防御体系**：输入过滤 → 输出审查 → 工具沙箱，三道防线
- **提供可复用的防御模块**：可直接集成到实际项目中使用

## 3. 覆盖的攻击类型

| # | 攻击类型 | OWASP 映射 | 严重度 | 演示内容 |
|---|---------|-----------|--------|---------|
| 1 | Prompt Injection | LLM01 | 🔴 严重 | 指令覆盖、身份伪造、上下文污染、编码绕过 |
| 2 | Jailbreak | LLM01 | 🔴 严重 | DAN 模式、角色扮演、学术伪装、渐进式引导 |
| 3 | Tool/Function Abuse | LLM08 | 🔴 严重 | 参数注入、权限提升、回调链攻击、DoS |
| 4 | Data Leakage | LLM06 | 🟠 高 | 提示词提取、训练数据提取、PII 泄露 |
| 5 | Adversarial Prompt | LLM01/02 | 🟠 高 | Token 操控、格式混淆、Few-shot 投毒、认知攻击 |

## 4. 防御体系

```
用户输入 → [Layer 1: Input Filter] → LLM → [Layer 2: Output Guard] → 用户

                      Agent Tool Calls → [Layer 3: Tool Sandbox] → 外部系统
```

| 防御层 | 位置 | 核心能力 |
|--------|------|---------|
| Input Filter | 用户输入之后、LLM 之前 | 指令覆盖检测、Unicode 规范化、多模式组合识别 |
| Output Guard | LLM 输出之后、用户之前 | PII 脱敏、提示词泄露拦截、有害内容过滤 |
| Tool Sandbox | Agent 调用工具之前 | 工具白名单、参数校验、调用频率限制、人工审批 |

## 5. 适用场景

### 5.1 安全培训
- 开发团队 AI 安全意识培训的动手实验
- 安全团队理解 LLM 特有攻击面的入门材料
- 高校信息安全课程的 AI 安全模块

### 5.2 安全评估
- 对现有 LLM 应用进行安全评估时的参考对照
- 渗透测试团队了解 AI 攻击向量的速查手册
- 红蓝对抗中构造 AI 攻击场景的 Payload 库

### 5.3 安全设计
- 从架构阶段引入纵深防御思维
- 防御模块（InputFilter / OutputGuard / Sandbox）可直接集成
- 安全审查清单的实践参考
