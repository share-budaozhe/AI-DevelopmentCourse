布道者

## 快速开始

### 零配置体验（模拟模式）

```bash
# 不需要 API Key，直接运行
python main.py --demo        # 演示模式：自动执行一个示例目标
python main.py               # 交互模式：从菜单选择目标
```

### LLM 驱动模式

```bash
# 1. 配置 API Key
cp .env.example .env
# 编辑 .env 填入 OPENAI_API_KEY=sk-xxxxx

# 2. 运行
python main.py --goal "你的目标" --live
```

## 运行模式详解

| 命令 | 说明 | 需要 API Key |
|------|------|:--:|
| `python main.py` | 交互模式：菜单选择目标 | 否 |
| `python main.py --demo` | 演示模式：预置演示目标 | 否 |
| `python main.py --goal "xxx"` | 直接指定目标（模拟推理） | 否 |
| `python main.py --goal "xxx" --live` | 真实 LLM 驱动 | 是 |
| `python main.py --goal "xxx" --max-iter 15` | 自定义迭代次数 | 否 |
| `python main.py --goal "xxx" --resume` | 从检查点恢复 | 否 |
| `python main.py --list-tools` | 列出可用工具 | 否 |
| `python main.py --config` | 显示当前配置 | 否 |

## 自定义目标

输入 `0` 进入自定义目标模式：

```bash
请输入你的目标（可以详细描述）:
> 帮我调研 LangChain 的最新版本 (v0.3+) 中的核心变更，
> 包括 RAG 管道的改进、Agent 框架的重构，
> 以及从 v0.2 迁移到 v0.3 的最佳实践。
```

更具体的目标 → 更精准的计划 → 更好的执行结果。

## 模拟模式 vs LLM 模式

| 维度 | 模拟模式 | LLM 模式 |
|------|---------|---------|
| API Key | 不需要 | 必需 |
| 推理质量 | 基于规则，可预测 | LLM 驱动，更智能 |
| 执行速度 | 快（纯本地） | 受 API 延迟影响 |
| 适用场景 | 学习架构、快速演示 | 真实任务执行 |
| 费用 | 免费 | 消耗 API 额度 |

**布道者建议**: 先用模拟模式理解 Agent 的执行流程，再用 LLM 模式处理真实任务。

## 最佳实践

### 1. 目标描述要具体

```
❌ 差: "帮我研究点东西"
✅ 好: "调研 Python asyncio 在 Web 框架中（FastAPI/Sanic）的性能优化策略，
       比较两者的异步 I/O 模式，并给出选型建议"

❌ 差: "写个程序"
✅ 好: "编写一个 Python CLI 工具，将 Markdown 文件批量转换为
       HTML，支持自定义 CSS 模板和代码高亮"
```

### 2. 合理设置迭代次数

```
简单任务（1-2 步）:   max_iter = 3-5
中等任务（3-5 步）:   max_iter = 8-12
复杂任务（5+ 步）:    max_iter = 15-20
```

### 3. 检查执行日志

Agent 执行完毕后，查看 `workspace/checkpoint_*.json` 了解详细执行过程。

### 4. 启用工具组合

在 `.env` 中开启所有工具以获得最佳体验：

```bash
AGENT_ALLOW_CODE_EXECUTION=true
AGENT_ALLOW_FILE_OPERATIONS=true
```
