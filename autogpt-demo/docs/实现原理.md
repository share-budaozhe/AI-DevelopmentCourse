布道者

## 1. 什么是 AutoGPT

AutoGPT 是由 Significant Gravitas 于 2023 年开源的项目，核心思想是**让 LLM 成为自主 Agent**——不只是回答问题，而是自主拆解任务、调用工具、评估结果、迭代执行，直到目标达成。

本 Demo 实现了 AutoGPT 的核心架构，将其简化为约 1000 行可读的 Python 代码。

## 3. 核心循环：Plan → Think → Act → Observe

布道者: 这是整个 AutoGPT 的精髓，一个持续迭代的自主推理循环。

### 3.1 Plan（规划阶段）

```
用户输入目标
    │
    ▼
┌──────────────────┐
│  分析目标语义      │  → 关键词提取 + 模式匹配
│  参考可用工具      │  → 列出可调用的能力
│  生成任务计划      │  → [{title, description, tool, criteria}]
└──────────────────┘
    │
    ▼
 子任务列表 (3-5 个)
```

**实现方式**：

1. **LLM 模式**: 将目标 + 工具描述发给 LLM，请求 JSON 格式的任务计划
2. **规则模式**: 基于目标关键词（调研/开发/数据）匹配预定义的模板

### 3.2 Think（思考阶段）

```
当前状态 (WorkingMemory)
    │
    ▼
┌──────────────────┐
│  回顾历史记忆      │  → 从 ShortTermMemory 获取最近 N 轮对话
│  参考长期经验      │  → 从 LongTermMemory 语义搜索相关知识
│  分析当前进度      │  → 检查哪些子任务已完成，哪些卡住了
│  推理下一步       │  → 结合目标和工具能力决定行动
└──────────────────┘
    │
    ▼
输出: THOUGHT + ACTION + ARGUMENTS
```

### 3.3 Act（行动阶段）

```
ACTION: web_search
    │
    ▼
┌──────────────────┐
│ ToolRegistry     │  → 查找工具
│ tool.execute()   │  → 执行操作
│ 返回 ToolResult   │  → {success, content, metadata}
└──────────────────┘
```

### 3.4 Observe（观察阶段）

```
ToolResult
    │
    ▼
┌──────────────────┐
│  分析执行结果      │  → 成功/失败？
│  更新 WorkingMem  │  → log_action()
│  更新 ShortTerm   │  → add_agent()
│  判断是否继续      │  → 子任务完成？/ 达到迭代上限？
└──────────────────┘
    │
    ├── 继续 → 回到 Think
    └── 完成 → Summary
```

## 4. 记忆系统设计

### 4.1 三层记忆架构

```
┌─────────────────────────────────────────────────────┐
│                  WorkingMemory                       │
│  (工作记忆 — 当前任务的临时状态)                       │
│  goal / plan[] / current_index / execution_log[]     │
│  生命周期: 单次任务，完成后清空                        │
├─────────────────────────────────────────────────────┤
│                  ShortTermMemory                     │
│  (短期记忆 — 当前任务的对话上下文)                      │
│  环形缓冲区, 最近 20 轮对话                            │
│  生命周期: 单次任务，自动淘汰旧内容                     │
├─────────────────────────────────────────────────────┤
│                  LongTermMemory                      │
│  (长期记忆 — 跨任务的知识积累)                         │
│  ChromaDB 向量存储, 语义搜索                           │
│  生命周期: 持久化，跨会话                              │
└─────────────────────────────────────────────────────┘
```

### 4.2 记忆读写流程

```
写流程:
  每次 Action 后:
    WorkingMemory.log_action()  ← 执行日志
    ShortTermMemory.add()       ← 对话上下文
  任务完成后:
    LongTermMemory.store()      ← 任务总结 + 经验

读流程:
  Think 阶段:
    ShortTermMemory.get_recent(10)    ← 当前上下文
    LongTermMemory.search(goal)       ← 历史经验
    WorkingMemory.get_progress()      ← 进度状态
```

## 5. 工具调用机制

### 工具基类设计

```python
class BaseTool(ABC):
    name: str           # 唯一标识（LLM 通过此名称调用）
    description: str    # 功能说明（LLM 据此判断何时使用）
    parameters: Dict    # 参数 Schema（JSON Schema 格式）

    @abstractmethod
    def execute(self, **kwargs) -> ToolResult:
        ...

class ToolRegistry:
    def register(tool):      # 注册工具
    def execute(name, **):   # 统一调用入口
    def get_commands_description():  # 生成 LLM 可读的命令列表
```

### 思考→行动 的解析

```
LLM 输出:
  THOUGHT: 我需要先搜索相关信息
  ACTION: web_search
  ARGUMENTS: {"query": "Python asyncio 最佳实践", "max_results": 5}

解析流程:
  1. 正则提取 ACTION: (\w+)
  2. 正则提取 ARGUMENTS: ({.*})
  3. JSON.parse(ARGUMENTS)
  4. ToolRegistry.execute(action, **args)
```

## 7. 与真实 AutoGPT 的对比

| 维度 | AutoGPT (原版) | 本 Demo |
|------|---------------|---------|
| Agent 循环 | Plan→Execute | Plan→Think→Act→Observe |
| 命令系统 | 20+ 命令 | 4 个核心指令 + finish |
| 记忆 | Pinecone/Weaviate/Redis | ChromaDB + 本地降级 |
| 插件系统 | 有（可扩展） | 内置四大工具 |
| 代码量 | ~10,000+ 行 | ~1,000 行 |
| 学习目标 | 生产使用 | 理解架构原理 |
| 无 API Key | 不支持 | ✅ 模拟模式 |

## 8. 设计模式

| 模式 | 应用位置 | 说明 |
|------|---------|------|
| **策略模式** | ToolRegistry → BaseTool | 可插拔的工具集 |
| **模板方法** | AutoGPTAgent.run() | 四阶段固定流程，细节可替换 |
| **观察者模式** | WorkingMemory.log_action() | 每次行动自动记录 |
| **责任链** | _plan_with_llm → _plan_with_rules | LLM 失败时自动降级 |
| **环形缓冲** | ShortTermMemory | 固定容量，自动淘汰 |
| **工厂方法** | _decide_action / _think | 根据模式（LLM vs 模拟）生成策略 |
