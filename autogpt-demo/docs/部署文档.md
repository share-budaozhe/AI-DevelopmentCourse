## 快速部署

### 方式一：本地安装

```bash
# 1. 克隆项目
cd autogpt-demo

# 2. 创建虚拟环境（推荐）
python -m venv venv

# 3. 激活虚拟环境
# Linux/macOS:
source venv/bin/activate
# Windows:
venv\Scripts\activate

# 4. 安装核心依赖
pip install -r requirements.txt

# 5. (可选) 配置 API Key
cp .env.example .env
# 编辑 .env，填入 OPENAI_API_KEY

# 6. 运行
python main.py --demo
```

### 方式二：Docker 部署

```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .

RUN mkdir -p workspace
VOLUME ["/app/workspace"]

CMD ["python", "main.py", "--demo"]
```

```bash
# 构建镜像
docker build -t autogpt-demo .

# 运行
docker run -it --rm -v ./workspace:/app/workspace autogpt-demo

# 或使用 docker-compose
cat > docker-compose.yml << EOF
services:
  autogpt:
    build: .
    volumes:
      - ./workspace:/app/workspace
    environment:
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    stdin_open: true
    tty: true
EOF

docker-compose up
```

## 配置说明

### 环境变量

编辑 `.env` 文件配置 Agent 行为：

```bash
# ══ LLM 配置 ══
OPENAI_API_KEY=sk-your-key-here       # 仅 --live 模式需要
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o

# ══ Agent 配置 ══
AGENT_NAME=AutoGPT-Demo               # Agent 名称
AGENT_MAX_ITERATIONS=10               # 最大循环次数
AGENT_MAX_TOKENS=4096                 # 每次 LLM 调用的 Token 上限
AGENT_TEMPERATURE=0.5                 # 输出随机性（0=确定，1=随机）

# ══ 工具开关 ══
AGENT_ALLOW_CODE_EXECUTION=true       # 是否允许代码执行
AGENT_ALLOW_FILE_OPERATIONS=true      # 是否允许文件操作

# ══ 记忆配置 ══
MEMORY_BACKEND=local                  # local / chroma
CHROMA_PERSIST_DIR=./workspace/chroma_db

# ══ 工作目录 ══
WORKSPACE_DIR=./workspace
```

### 配置建议

| 场景 | temperature | max_iterations | 说明 |
|------|-----------|----------------|------|
| 精确任务（代码/计算） | 0.2-0.3 | 5-10 | 低温度保证一致性 |
| 创意任务（调研/写作） | 0.5-0.7 | 10-15 | 适度随机激发创意 |
| 探索任务（研究/分析） | 0.4-0.6 | 15-20 | 平衡精确与探索 |

## 故障排查

### 问题：找不到模块

```
ModuleNotFoundError: No module named 'xxxxx'
```

**解决**: 确保在项目根目录执行，或设置 PYTHONPATH:

```bash
# Linux/macOS
export PYTHONPATH="${PYTHONPATH}:$(pwd)"
# Windows
set PYTHONPATH=%PYTHONPATH%;%cd%

# 然后重试
python main.py --demo
```

### 问题：venv 激活后 pip 不可用

```bash
python -m ensurepip --upgrade
python -m pip install -r requirements.txt
```

### 问题：ChromaDB 安装失败

ChromaDB 在 Windows 上可能需要 Visual C++ Build Tools:

```bash
# 方案一: 安装 Build Tools
# 下载: https://visualstudio.microsoft.com/visual-cpp-build-tools/

# 方案二: 跳过 ChromaDB（记忆降级为本地模式）
pip install -r requirements.txt --no-deps chromadb
# 长期记忆自动降级，不影响核心功能
```
