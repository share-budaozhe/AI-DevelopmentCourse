# 向量嵌入（Embeddings）入门

## 什么是向量嵌入？

向量嵌入是把文字、图片等非结构化数据转换为一串数字（向量）的技术。
语义相似的文本，它们的向量在空间中距离很近。

例如：
- "猫在沙发上睡觉" 和 "小猫蜷在沙发里打盹" → 向量距离很近
- "猫在沙发上睡觉" 和 "今天股市大涨" → 向量距离很远

## 嵌入模型

常用的嵌入模型：
- OpenAI text-embedding-3-small（1536 维）
- OpenAI text-embedding-3-large（3072 维）
- BGE（BAAI General Embedding）
- Cohere Embed
- 本地开源模型（如 all-MiniLM-L6-v2）

## 向量检索原理

1. 将查询文本转为向量 Q
2. 将文档库中所有文本块转为向量 D1, D2, ..., Dn
3. 计算 Q 与每个 D 的余弦相似度
4. 返回相似度最高的 Top-K 个文档块

## 相似度计算方法

- 余弦相似度（Cosine Similarity）：最常用，范围 [-1, 1]
- 欧氏距离（Euclidean Distance）：范围 [0, ∞)
- 点积（Dot Product）：范围 (-∞, ∞)

## 为什么向量检索有效？

它捕捉的是"语义相似"而非"关键词匹配"。
即使两个文本没有共同的词，只要意思相近，它们的向量距离就会很近。
这让 LLM 能"理解"问题，而不是做简单的字符串匹配。
