# RAG（检索增强生成）完全指南

## 什么是 RAG？

RAG（Retrieval-Augmented Generation）是一种让 LLM 在生成回答前先从外部知识库检索相关信息的架构。
它的核心思想是：不给 LLM 灌输所有知识（不可能也不经济），而是让它在需要时"查资料"。

## RAG 的标准流程

1. **加载（Loading）**：从各种数据源读取文档（PDF、网页、数据库等）
2. **解析（Parsing）**：将文档拆分为合适的文本块（Chunk/Node）
3. **索引（Indexing）**：对文本块生成向量嵌入，存入向量数据库
4. **检索（Retrieval）**：用户提问时，从向量库中找到最相关的文本块
5. **合成（Synthesis）**：将检索到的文本块与用户问题一起发给 LLM，生成最终答案

## 为什么需要 RAG？

1. **知识时效性**：LLM 的训练数据有截止日期，RAG 可以接入最新信息
2. **幻觉问题**：RAG 让 LLM 基于真实文档回答，大幅减少编造
3. **领域专业性**：通用 LLM 不懂你的私有文档，RAG 让它"读"你的文档
4. **可解释性**：RAG 能告诉你答案来源于哪篇文档的哪个片段
5. **成本效率**：不需要微调（fine-tuning），索引一次即可反复查询

## 进阶方向

- 多跳检索（Multi-hop Retrieval）
- 递归检索（Recursive Retrieval）
- 混合检索（Hybrid Search：向量 + 关键词）
