## 2. 状态流转图

```
初始化 (initialized)
    │
    ▼
研究中 (researching) ──→ 研究完成 (research_completed)
                                        │
                                        ▼
                                   分析中 (analyzing) ──→ 分析完成 (analysis_completed)
                                                              │
                                                              ▼
                                                         写作中 (writing) ──→ 写作完成 (writing_completed)
                                                                                  │
                                                                                  ▼
                                                                             审核中 (reviewing) ──→ 审核完成 (review_completed)
                                                                                                        │
                                                                                                        ▼
                                                                                                   汇总中 (summarizing) ──→ 已完成 (completed)

在任何阶段发生错误时: ──→ 错误 (error) ──→ 流程终止
```

## 3. Agent 内部执行流程

每个 Agent 的内部执行逻辑一致，以研究 Agent 为例：

```
┌──────────────────────────────────────────┐
│              Research Agent               │
│                                          │
│  1. 接收课题 topic                        │
│  2. 写入共享上下文 topic                  │
│  3. 更新状态 → researching                │
│  4. 构建消息 (System Prompt + User Msg)   │
│  5. 调用 LLM（绑定工具）                   │
│     │                                    │
│     ├─ 有工具调用？ → 执行工具 → 继续对话   │
│     │                                    │
│     └─ 无工具调用？ → 直接获取回复         │
│  6. 写入研究笔记到共享上下文               │
│  7. 更新状态 → research_completed         │
│  8. 返回结果                              │
└──────────────────────────────────────────┘
```
