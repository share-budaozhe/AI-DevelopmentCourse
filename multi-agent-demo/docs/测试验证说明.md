# 多智能体协作系统 —— 测试与验证说明

## 1. 测试策略

本项目的测试采用 **分层测试策略**，从底层工具到上层工作流逐步覆盖：

```
         ┌──────────────────┐
         │   集成测试 (E2E)  │  ← 验证完整工作流
         └──────────────────┘
                    ↑
         ┌──────────────────┐
         │  工作流测试       │  ← 验证 LangGraph 编排
         └──────────────────┘
                    ↑
         ┌──────────────────┐
         │  Agent 单元测试   │  ← 验证各 Agent 执行逻辑
         └──────────────────┘
                    ↑
         ┌──────────────────┐
         │  工具函数测试     │  ← 验证工具和共享上下文
         └──────────────────┘
```

## 2. 测试文件结构

```python
tests/
├── __init__.py
└── test_agents.py    # 所有测试用例
```

## 3. 测试用例详解

### 3.1 工具层测试（无需 API Key）

这些测试验证核心工具函数的正确性，不依赖外部服务。

| 测试类 | 测试用例 | 验证内容 | 断言 |
|--------|---------|---------|------|
| TestTools | `test_search_knowledge_base` | 知识库检索功能 | 返回内容包含查询词，长度>20 |
| TestTools | `test_search_knowledge_base_no_match` | 无匹配时的处理 | 返回"未找到"提示 |
| TestTools | `test_read_write_shared_context` | 共享上下文读写 | 写入后能正确读出 |
| TestTools | `test_read_nonexistent_field` | 读取不存在的字段 | 返回"不存在"提示 |
| TestTools | `test_write_invalid_field` | 写入无效字段 | 返回"不存在"提示 |
| TestTools | `test_calculate_addition` | 数学加法 | 结果=5 |
| TestTools | `test_calculate_complex` | 复杂表达式 | 结果=7 |
| TestTools | `test_calculate_error` | 错误表达式处理 | 返回"错误"提示 |
| TestTools | `test_shared_context_status` | 状态字段流转 | 状态值正确变更 |

### 3.2 配置测试（无需 API Key）

| 测试用例 | 验证内容 |
|---------|---------|
| `test_get_llm_default` | LLM 实例能正常创建 |
| `test_get_llm_with_params` | 带参数创建 LLM 实例 |

### 3.3 工作流测试（无需 API Key）

| 测试用例 | 验证内容 |
|---------|---------|
| `test_workflow_graph_creation` | LangGraph 状态图能正常构建 |
| `test_workflow_state_structure` | AgentState 数据结构正确 |

### 3.4 集成测试（需要 API Key）

这些测试使用真实 LLM API，通过 `@pytest.mark.skipif` 条件跳过。

| 测试用例 | 验证内容 | 备注 |
|---------|---------|------|
| `test_full_workflow` | 完整工作流执行 | 默认跳过（耗时长） |
| `test_single_agent_research` | 单个研究 Agent | 快速验证 Agent 可用性 |

## 4. 运行测试

```bash
# 安装测试依赖
pip install pytest

# 运行所有单元测试（无需 API Key）
pytest tests/ -v -k "not Integration"

# 运行含集成测试（需要配置 API Key）
pytest tests/ -v

# 仅运行工具测试
pytest tests/ -v -k "TestTools"

# 仅运行集成测试
pytest tests/ -v -k "Integration"

# 生成测试覆盖率报告
pip install pytest-cov
pytest tests/ -v --cov=src --cov-report=term-missing
```

## 5. 测试环境准备

### 5.1 前置条件

```bash
# 1. 安装项目依赖
pip install -r requirements.txt

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env，填入 API Key（集成测试需要）
```

### 5.2 CI/CD 集成示例

```yaml
# .github/workflows/test.yml
name: Test Multi-Agent Demo

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install pytest pytest-cov
      - name: Run unit tests
        run: |
          pytest tests/ -v -k "not Integration"
          # 集成测试仅在 main 分支运行
      - name: Run integration tests
        if: github.ref == 'refs/heads/main'
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
        run: |
          pytest tests/ -v
```

## 6. 验证清单

### 6.1 功能验证

| 检查项 | 预期结果 | 验证方法 |
