RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索与文本生成的技术架构。它通过从外部知识库中检索相关文档片段，并将其作为上下文提供给大语言模型，从而生成更准确、更具时效性的回答。

RAG的核心工作流程：
1. 索引阶段（离线）：将文档分割成小块(chunk)，使用嵌入模型将每个块转换为向量，存入向量数据库。
2. 检索阶段（在线）：将用户查询同样转换为向量，在向量数据库中检索最相似的Top-K个文档块。
3. 生成阶段：将检索到的文档块与用户查询拼接成提示词，发送给大语言模型生成最终答案。

RAG相比纯生成模型的优势：
- 减少了幻觉(Hallucination)问题：模型基于真实文档回答问题
- 知识可更新：只需更新知识库，无需重新训练模型
- 可溯源：答案可以追溯到具体的来源文档
- 领域适配：可以针对特定领域构建专属知识库

常见的RAG实现方案：
- LangChain：提供了完整的RAG链式调用框架
- LlamaIndex：专注于数据索引和检索
- Haystack：面向生产环境的NLP框架
- 自建方案：Embedding模型 + 向量数据库 + LLM API

向量数据库的选择包括ChromaDB（轻量级）、Pinecone（云服务）、Weaviate、Milvus、FAISS等。嵌入模型常用的有OpenAI的text-embedding-ada-002、sentence-transformers系列、BGE系列等。

RAG的进阶技术包括：查询重写、HyDE（假设性文档嵌入）、父子文档检索、多路召回与融合排序等。
