# 安全沙箱应用 Demo —— 安全策略设计

## 概述

本 Demo 实现了多层安全策略体系，包含：

```
代码输入
  │
  ├─→ Layer 0: 输入验证 (BaseSandbox._validate_input)
  │    └── 空代码检测 / 长度限制 (500KB)
  │
  ├─→ Layer 1: AST 静态分析 (CodeSecurityAnalyzer)
  │    ├── AST 语法结构检查
  │    ├── 导入白名单/黑名单
  │    ├── 危险函数调用检测
  │    └── 注入模式识别
  │
  ├─→ Layer 2: 沙箱执行 (BaseSandbox._execute_impl)
  │    ├── Subprocess 沙箱 / Docker 沙箱 / RestrictedPython
  │    └── OS 级资源限制 (RLIMIT / Cgroup)
  │
  ├─→ Layer 3: 资源监控 (ResourceMonitor)
  │    ├── 内存/CPU/文件描述符/线程
  │    └── 超限告警回调
  │
  ├─→ Layer 4: 输出处理 (BaseSandbox.execute)
  │    ├── 输出截断 (max_output_chars)
  │    ├── 结果序列化
  │    └── 审计日志记录
  │
  └─→ 审计
       └── 完整审计日志: id + timestamp + sandbox + action + detail
```

## 安全等级

| 等级 | 导入策略 | 内置函数 | 允许的 AST 节点 | 适用场景 |
|------|---------|---------|----------------|---------|
| **STRICT** | 白名单 (~20 模块) | 安全子集 (无 I/O) | 基础语法（无 class/async） | 教学平台、考试系统 |
| **STANDARD** | 黑名单 (禁止危险模块) | 基础 + 安全 I/O | 大部分 Python 语法 | Agent 代码执行、计算任务 |
| **RELAXED** | 仅过滤明确危险 | 大部分可用 | 全部语法 | 可信用户、开发环境 |

## 安全规则详解

### 禁止导入的模块 (STANDARD 级别)

```python
FORBIDDEN_MODULES_STANDARD = {
    "os",          # 操作系统接口（文件/进程/环境变量）
    "subprocess",  # 子进程启动
    "shutil",      # 高级文件操作
    "socket",      # 网络 socket
    "requests",    # HTTP 请求
    "pickle",      # 反序列化（代码执行漏洞）
    "marshal",     # 序列化
    "ctypes",      # 调用 C 动态库
    "cffi",        # 调用 C 动态库
    "multiprocessing",  # 多进程
    "signal",      # 信号处理
}
```

### 禁止的内置函数 (STANDARD 级别)

```python
FORBIDDEN_BUILTINS = {
    "eval",        # 执行任意 Python 表达式
    "exec",        # 执行任意 Python 语句
    "compile",     # 编译代码对象
    "__import__",  # 底层导入函数
    "open",        # 文件操作
    "input",       # 用户交互
    "globals",     # 全局命名空间访问
    "locals",      # 本地命名空间访问
    "vars",        # 对象属性字典
    "getattr",     # 动态属性访问
    "setattr",     # 动态属性修改
    "delattr",     # 动态属性删除
    "breakpoint",  # 调试器断点
}
```

### 危险 AST 节点

```
禁止的 AST 节点 (STRICT):
    Import          — import 语句
    ImportFrom      — from ... import 语句
    Global          — global 变量声明
    Nonlocal        — nonlocal 变量声明
    ClassDef        — class 定义
    AsyncFunctionDef — async def
    Await / Yield / YieldFrom
```

### 攻击模式检测

```python
# Shell 注入
r"rm\s+-rf\s+/"           # 递归根目录删除
r"curl\s+.*\|\s*(ba)?sh"  # 远程脚本管道执行
r"wget\s+.*-O\s*-"        # 下载到管道

# 路径遍历
r"\.\.\/\.\.\/"           # 目录穿越

# 代码混淆
r"chr\(\s*\d+\s*\)\s*\.\s*join"  # chr join 混淆
r"base64\..*decode"              # base64 解码
```

## 安全边界与威胁模型

### 信任边界

```

│              不受信任区域 (Untrusted)                │
│  ┌──────────┐  ┌──────────┐  ┌──────────────────┐ │
│  │ 用户代码  │  │ LLM 输出 │  │ 第三方上传代码   │ │
│  └─────┬────┘  └─────┬────┘  └────┬─────────────┘ │
│        │              │            │                │
│  ─ ─ ─ ┼ ─ ─ ─ ─ ─ ─ ┼ ─ ─ ─ ─ ─ ┼ ─ ─ ─ ─ ─ ─ │
│        │              │            │  安全边界      │
│        ▼              ▼            ▼                │
│  ┌─────────────────────────────────────────────┐   │
│  │              沙箱层                           │   │
│  │  AST 分析 → 策略检查 → 资源限制 → 审计         │   │
│  └─────────────────────────────────────────────┘   │
│                         │                           │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┼ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ │
│                         │         安全边界          │
└─────────────────────────┼──────────────────────────┘

┌─────────────────────────┼──────────────────────────┐
│          受信任区域 (Trusted)  │                      │
│                         ▼                           │
│  ┌─────────────────────────────────────────────┐   │
│  │              Host System                      │   │
│  │  文件系统 / 网络 / 数据库 / 系统资源           │   │
│  └─────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────┘
```

### 威胁场景与应对

| 威胁 | 风险等级 | 应对措施 |
|------|---------|---------|
| 读取系统文件 (如 /etc/passwd) | 高 | 禁止 open / os 模块 |
| 网络外连 (数据外泄) | 高 | Docker: network_mode=none |
| Fork 炸弹 (耗尽进程) | 高 | RLIMIT_NPROC / pids_limit |
| 内存炸弹 (OOM) | 高 | RLIMIT_AS / mem_limit |
| 死循环 (CPU 耗尽) | 中 | CPU time limit / 超时 kill |
| 磁盘写满 | 中 | Docker: read_only rootfs + tmpfs |
| Pickle 反序列化攻击 | 严重 | 禁止 pickle 导入 |
| Python 沙箱逃逸 (__subclasses__) | 严重 | 禁止 getattr / __class__ |
| 侧信道攻击 (时间/内存) | 低 | 统一资源限制、返回统一超时消息 |

## 审计日志

每次代码执行都记录结构化审计日志：

```json
{
  "id": "a1b2c3d4",
  "timestamp": "2025-07-01T14:30:00",
  "sandbox_type": "SubprocessSandbox",
  "action": "FAILED",
  "detail": "安全策略检查未通过 (risk=85): 禁止导入危险模块: os"
}
```

安全团队可以通过审计日志：
- 追踪所有代码执行尝试
- 识别攻击模式和频率
- 调优安全策略规则
- 满足合规要求

## 集成到 AI Agent 的示例

```python
from src.sandboxes.base import SandboxConfig
from src.sandboxes.docker_sandbox import DockerSandbox

class SecureAgentExecutor:
    def __init__(self):
        self.sandbox = DockerSandbox(
            SandboxConfig(
                timeout_seconds=10,
                memory_limit_mb=256,
                allow_network=False,
                allow_subprocesses=False,
                allow_file_write=False,
            )
        )

    def execute_code(self, code: str, task_id: str) -> dict:
        result = self.sandbox.execute(code)
        return {
            "task_id": task_id,
            "success": result.success,
            "stdout": result.stdout,
            "error": result.error,
            "audit_id": result.audit_id,
        }
```
