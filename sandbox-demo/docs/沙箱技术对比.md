# 安全沙箱应用 Demo —— 沙箱技术对比

## 核心指标对比

| 维度 | Subprocess 沙箱 | Docker 容器沙箱 | RestrictedPython |
|------|:--:|:--:|:--:|
| **隔离级别** | 进程级 🟡 | 容器级 🟢 | 语言级 🟠 |
| **启动速度** | ~50ms | ~1s | < 1ms |
| **内存额外开销** | ~30MB | ~50MB | ~5MB |
| **CPU 限制** | RLIMIT_CPU ✅ | Cgroup ✅ | SIGALRM 🟡 |
| **内存限制** | RLIMIT_AS ✅ | Cgroup ✅ | 无 🟠 |
| **网络隔离** | 无 | network_mode=none | 无 |
| **文件系统隔离** | 无 | read_only rootfs | builtins 替换 |
| **进程数限制** | RLIMIT_NPROC ✅ | pids_limit ✅ | 无 |
| **外部依赖** | 无 ✅ | Docker ⚠️ | 无 ✅ |
| **跨平台** | ✅ | ⚠️ 需 Docker | ✅ |
| **绕过难度** | 中 | 极高 | 中-高 |

## 各沙箱详细介绍

### 1. Subprocess 资源限制沙箱

**隔离机制**：
- `subprocess.Popen` 启动独立子进程
- `resource.setrlimit()` 设置 OS 级硬限制
- `signal.SIGALRM` 墙上时钟超时
- `signal.SIGXCPU` CPU 时间超时
- `sys.modules` 移除危险模块

**资源限制项**：

| 限制项 | RLIMIT 常量 | 默认值 |
|--------|------------|--------|
| CPU 时间 | RLIMIT_CPU | 10 秒 |
| 地址空间 | RLIMIT_AS | 256 MB |
| 数据段 | RLIMIT_DATA | 136 MB |
| 进程数 | RLIMIT_NPROC | 0 (禁止子进程) |
| 文件大小 | RLIMIT_FSIZE | 100 MB |
| 文件描述符 | RLIMIT_NOFILE | 256 |

**局限**：
- 不是真正的内核级隔离
- RLIMIT_AS 在部分 Linux 发行版上行为不一致
- 无法阻止容器逃逸级别的攻击
- Windows 上部分 RLIMIT 不可用

**适用场景**：
- 中等安全要求的代码执行
- LLM Agent 生成的代码片段
- 可信用户的批量任务

### 2. Docker 容器沙箱

**隔离机制**（8 层纵深防御）：

```
┌─────────────────────────────────────┐
│ Layer 1: Linux Namespace            │
│ PID/NET/MNT/IPC/UTS 隔离             │
├─────────────────────────────────────┤
│ Layer 2: Cgroup                     │
│ cpu_quota + mem_limit + pids_limit  │
├─────────────────────────────────────┤
│ Layer 3: Linux Capabilities         │
│ cap_drop=ALL + no-new-privileges    │
├─────────────────────────────────────┤
│ Layer 4: 只读文件系统                │
│ read_only=True + tmpfs /tmp         │
├─────────────────────────────────────┤
│ Layer 5: 网络隔离                   │
│ network_mode=none                   │
├─────────────────────────────────────┤
│ Layer 6: Docker Ulimits             │
│ nproc / nofile / fsize              │
├─────────────────────────────────────┤
│ Layer 7: Python 级 RLIMIT           │
│ 容器内的第二道资源防线               │
├─────────────────────────────────────┤
│ Layer 8: Auto-remove                │
│ 执行后自动销毁，不留痕迹             │
└─────────────────────────────────────┘
```

**Docker Run 参数详解**：

```python
container = client.containers.run(
    image="python:3.11-slim",
    command=["python", "-c", code],
    detach=True,
    # Cgroup 资源限制
    mem_limit="256m",            # 硬内存上限
    memswap_limit="512m",        # swap 上限 (mem+swap)
    cpu_quota=50000,             # CPU 配额 (微秒/周期)
    cpu_period=100000,           # CPU 周期 (微秒)
    pids_limit=32,               # 最大进程数
    # 安全加固
    network_mode="none",          # 禁用网络
    read_only=True,               # 只读根文件系统
    security_opt=["no-new-privileges:true"],  # 禁止提权
    cap_drop=["ALL"],             # 移除所有内核能力
    # Ulimits
    ulimits=[
        Ulimit(name='nproc', soft=1, hard=32),
        Ulimit(name='nofile', soft=64, hard=256),
    ],
    # 临时文件
    tmpfs={"/tmp": "size=64M,noexec,nosuid"},
    # 自动销毁
    auto_remove=True,
)
```

**局限**：
- 需要 Docker 环境
- 启动延迟较高（~1s）
- 内存开销较大（~50MB + 基础镜像）
- Windows 上行为略有差异

**适用场景**：
- 高安全要求的代码执行
- 不受信任的第三方代码
- 多租户 SaaS 平台

### 3. RestrictedPython AST 沙箱

**隔离机制**：
- AST 解析 → 变换 → 安全内置函数替换
- 导入白名单（STRICT 模式仅允许 ~20 个安全模块）
- 属性访问控制（禁止 `__class__` 等特殊属性）

**安全等级**：

| 模式 | 允许导入 | 允许内置函数 | 允许语法 |
|------|---------|-------------|---------|
| STRICT | 仅白名单 (~20 模块) | 安全子集 (无 open/print 等) | 仅基础语法 |
| STANDARD | 黑名单过滤 | 基础 + 安全 I/O | 大部分语法 |
| RELAXED | 仅过滤明确危险 | 大部分可用 | 全部语法 |

**STRICT 模式允许的模块**：
```
math, cmath, decimal, fractions
random, statistics
json
datetime, calendar
collections, itertools, functools
re, string, textwrap
typing, dataclasses
hashlib, base64
```

**安全内置函数替换**：

| 原始 | 替换 | 行为 |
|------|------|------|
| `__import__` | `_safe_import` | 仅允许白名单模块 |
| `open` | `_safe_open` | 直接抛 PermissionError |
| `getattr` | `_safe_getattr` | 禁止访问 `__xxx__` 属性 |
| `eval` / `exec` | 删除 | NameError |
| `compile` | 删除 | NameError |

**局限**：
- 不是真正的 OS 级隔离
- 对于 pickle 反序列化等攻击防御不足
- Python 语法子集受限
- 不支持类定义（STRICT 模式）

**适用场景**：
- 纯计算型任务
- 用户自定义公式
- 教学/考试平台
- 规则引擎

## 性能基准测试

| 操作 | Subprocess | Docker | RestrictedPython |
|------|:--:|:--:|:--:|
| 空代码执行 | ~45ms | ~800ms | ~0.3ms |
| 简单计算 (fib 20) | ~48ms | ~820ms | ~0.5ms |
| 导入 + 计算 | ~52ms | ~850ms | ~0.8ms |
| 批量 100 次执行 | ~5s | ~85s | ~0.05s |

> 注：Docker 首次执行时可能需 pull 镜像（+30s-2min），上表不含此时间。
